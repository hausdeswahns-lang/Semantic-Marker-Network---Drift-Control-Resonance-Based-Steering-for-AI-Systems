## 8. LTS-INP-T-V1 (INPUT TEMPLATES)

| Template-Name | Trigger-Wort | Funktion |
|:---|:---|:---|
| *'SMN-FIX-V6.3.1'* | FIX, FORMAT | Gibt die vollständige Formatstruktur für alle SMN-Ausgaben aus: nummerierte Abschnitte (1–8), Markdown-Tabellen, einfache Fettdrucklogik, kein LaTeX, optionaler Codeblock-Abschluss, Lizenzzeile am Ende.  
Begründung: Das SMN entsteht nativ in Deutsch (Entwicklung, Governance, Promptstruktur), aber CHANGELOGs sind strukturierte Export-Artefakte für internationale Plattformen. Die Trennung schützt semantische Nähe im Interface und gewährleistet strukturelle Sichtbarkeit im globalen Kontext. |
| *'SMN-CORE-LTS-PROMPT'* | CORE-PROMPT | Generiert den aktuellen SMN CORE PROMPT. |
| *'SMN-Audit-Full'* | AUDIT | Prüft SMN-Status, Drift Score, Kernel-Version. |
| *'SMN-EXPLAIN-PROMPT'* | LERNPROMPT | Generiert den vollständigen Erklär-Prompt. |
| *'SMN-EXPLAIN-SINGLE'* | LERNBLOCK | Gibt den gesamten Lerninhalt in einem Block aus. |
| *'SMN-CHANGELOG-V6.3.0'* | CHANGE | Generiert den strukturierten CHANGELOG (EN). |
| *'SMN-FEATURE-STATUS'* | FEATURE | Gibt den Status aller relevanten SMN-Kernfunktionen aus. |
| *'Code-Complexity-Check'* | COMPLEXITY | Analysiert Code auf Komplexität und Token-Effizienz. |
| *'System-Abstraktion'* | ABSTRAHIEREN | Reduziert technischen Text auf Kernfunktionen. |
| *'API-Schema-Verify'* | SCHEMA | Vergleicht zwei API-Schemas tabellarisch. |
| *'Seismic-Accel-V4-Flex'* | ERDBEBEN | Führt Bau-Gefahrenanalyse anhand von Geodaten durch. |
